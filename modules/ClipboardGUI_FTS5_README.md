# ClipboardGUI_FTS5 高性能剪贴板管理器 GUI

## 概述

这是一个高性能的剪贴板管理器 GUI 界面，支持大量数据的快速显示和搜索。

## 功能特性

- ✅ **双栏布局**：左侧列表 + 右侧详情显示
- ✅ **无边框搜索框**：顶部搜索输入框
- ✅ **暗黑模式配色**：现代化的深色主题
- ✅ **SourceApp 颜色标识**：在详情中显示不同应用的颜色标识
- ✅ **数据缓存**：使用内存缓存提升性能
- ✅ **搜索功能**：支持实时搜索过滤
- ✅ **双击复制**：双击列表项快速复制内容

## 使用方法

### 1. 在主脚本中引入模块

```autohotkey
#Include modules\ClipboardFTS5.ahk
#Include modules\ClipboardGUI_FTS5.ahk

; 初始化数据库和 GUI
InitClipboardFTS5DB()
InitClipboardFTS5GUI()
```

### 2. 绑定 CapsLock+X 热键

在主脚本的热键定义区域添加：

```autohotkey
#HotIf GetCapsLockState()

; X 键打开剪贴板管理器 GUI
x:: {
    ToggleClipboardFTS5GUI()
}

#HotIf
```

### 3. API 参考

#### ShowClipboardFTS5GUI()

显示剪贴板管理器 GUI。

**说明**：
- 如果 GUI 已显示，会聚焦到搜索框
- 首次显示时会自动加载数据

---

#### HideClipboardFTS5GUI()

隐藏剪贴板管理器 GUI。

---

#### ToggleClipboardFTS5GUI()

切换 GUI 的显示/隐藏状态。

---

#### LoadClipboardFTS5Data(keyword := "")

加载数据到缓存并更新列表显示。

**参数**：
- `keyword` (String, 可选)：搜索关键词。如果为空，加载所有数据（最多5000条）

**说明**：
- 数据从 `ClipboardFTS5DB` 数据库加载
- 如果提供了关键词，使用 FTS5 全文搜索
- 数据加载到 `ClipboardFTS5DisplayCache` 缓存中
- 列表会自动更新

---

## 性能优化

1. **数据缓存**：使用 `ClipboardFTS5DisplayCache` 内存数组缓存数据
2. **分页加载**：默认最多加载 5000 条记录，避免内存过载
3. **重绘优化**：更新列表时使用 `-Redraw` 和 `+Redraw` 减少闪烁
4. **FTS5 搜索**：使用 SQLite FTS5 全文搜索，搜索速度快

## 注意事项

1. **虚拟列表**：当前实现使用普通 ListView，对于超过 5000 条的数据，建议使用搜索功能或分页
2. **数据库依赖**：需要先初始化 `ClipboardFTS5DB` 数据库
3. **热键绑定**：CapsLock+X 热键需要在主脚本中绑定，因为需要 `#HotIf GetCapsLockState()` 上下文
4. **窗口大小**：窗口支持调整大小，布局会自动适应

## 扩展功能建议

如果需要对千万级数据支持，可以考虑：

1. **实现真正的虚拟列表**（+OwnerData + LVM_SETITEMCOUNT + LVN_GETDISPINFO）
2. **分页加载**：实现上一页/下一页按钮
3. **延迟加载**：仅在滚动到可视区域时加载数据
4. **数据库索引优化**：确保 Timestamp 字段有索引
