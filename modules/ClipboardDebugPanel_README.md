# 剪贴板调试面板使用说明

## 功能概述

`ClipboardDebugPanel.ahk` 是一个专业的调试工具，用于诊断剪贴板数据无法显示在历史面板中的问题。

## 主要功能

### 1. 实时状态监控
- **剪贴板监听器状态**：显示监听器是否已注册
- **数据库连接状态**：显示数据库是否已连接
- **最后剪贴板内容**：实时显示当前剪贴板内容

### 2. 统计信息
- **监听事件数**：记录所有剪贴板变化事件
- **成功保存数**：记录成功保存到数据库的次数
- **失败次数**：记录保存失败的次数

### 3. 详细调试日志
- 实时记录每个步骤的执行情况
- 显示时间戳（精确到毫秒）
- 记录所有错误和异常信息
- 自动滚动到最新日志

### 4. 测试功能
- **测试监听器**：检查剪贴板监听器是否正常工作
- **测试数据库**：检查数据库连接和查询功能
- **测试保存**：手动测试数据保存功能
- **清空日志**：清除所有调试日志

## 使用方法

### 自动启动（已集成）

在 `clipboard.ahk` 中已经自动包含并启动调试面板：

```ahk
#Include modules\ClipboardDebugPanel.ahk
InitClipboardDebugPanel()
ShowClipboardDebugPanel()  ; 自动显示调试面板
```

### 手动启动

如果需要单独使用调试面板：

```ahk
#Requires AutoHotkey v2.0
#Include modules\ClipboardDebugPanel.ahk

InitClipboardFTS5DB()
InitClipboardDebugPanel()
ShowClipboardDebugPanel()

Persistent
```

## 诊断流程

### 步骤 1：检查监听器
1. 查看"剪贴板监听器"状态，应该显示"已注册"（绿色）
2. 点击"测试监听器"按钮
3. 尝试复制一些文本（Ctrl+C）
4. 观察日志中是否出现"监听触发"事件

**如果看不到"监听触发"事件：**
- 检查脚本是否以管理员权限运行
- 检查是否有其他程序占用剪贴板监听
- 查看是否有错误信息

### 步骤 2：检查数据库
1. 查看"数据库连接"状态，应该显示"已连接"（绿色）
2. 点击"测试数据库"按钮
3. 查看日志中的测试结果

**如果数据库未连接：**
- 检查 `sqlite3.dll` 是否存在
- 检查数据库文件权限
- 查看错误信息

### 步骤 3：检查保存流程
1. 复制一些文本（Ctrl+C）
2. 观察日志中的完整流程：
   - `[监听触发]` - 监听器被触发
   - `[防抖]` - 防抖处理
   - `[处理开始]` - 开始处理
   - `[类型判断]` - 判断内容类型
   - `[内容检查]` - 检查内容
   - `[去重检查]` - 去重检查
   - `[数据库检查]` - 检查数据库
   - `[保存成功]` 或 `[保存失败]` - 保存结果
   - `[面板刷新]` - 刷新面板

**如果保存失败：**
- 查看错误信息
- 检查数据库表结构
- 检查内容是否包含特殊字符

### 步骤 4：测试保存功能
1. 点击"测试保存"按钮
2. 查看日志中的测试结果
3. 检查数据库中是否有新记录

## 常见问题诊断

### 问题 1：监听器未触发
**症状**：复制后没有任何日志
**可能原因**：
- 脚本未以管理员权限运行
- 其他程序占用剪贴板监听
- OnClipboardChange 未正确注册

**解决方法**：
- 以管理员权限运行脚本
- 关闭其他剪贴板管理工具
- 检查脚本初始化顺序

### 问题 2：数据库未连接
**症状**：数据库状态显示"未连接"（红色）
**可能原因**：
- sqlite3.dll 不存在
- 数据库文件被锁定
- 初始化失败

**解决方法**：
- 检查 sqlite3.dll 文件
- 关闭其他使用数据库的程序
- 重新初始化数据库

### 问题 3：保存失败
**症状**：日志显示"保存失败"
**可能原因**：
- 数据库表不存在
- 内容包含特殊字符
- 数据库写入权限不足

**解决方法**：
- 运行"测试数据库"检查表结构
- 查看具体错误信息
- 检查文件权限

### 问题 4：面板未刷新
**症状**：数据已保存但面板未更新
**可能原因**：
- 面板未打开（HistoryIsVisible = false）
- 刷新函数未调用
- 刷新延迟未执行

**解决方法**：
- 确保历史面板已打开
- 检查日志中的"面板刷新"记录
- 手动刷新面板

## 注意事项

1. 调试面板会记录所有操作，日志可能很长
2. 建议在诊断完成后关闭调试面板以提高性能
3. 日志最多保留200行，超出会自动删除最旧的
4. 统计信息会实时更新，反映当前会话的状态

## 性能影响

- 调试面板每1秒刷新一次状态
- 日志记录对性能影响极小（<1ms）
- 建议在诊断完成后关闭调试面板
