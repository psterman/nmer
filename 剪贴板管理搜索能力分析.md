## 二、SearchCenter 搜索能力

### 2.1 搜索架构

**多数据源并行搜索**：
- 剪贴板历史（ClipMain FTS5数据库）
- 提示词模板（Prompts表）
- 配置文件项（CursorShortcut.ini）
- **文件路径（Everything搜索）**
- 快捷键映射
- 功能函数
- UI标签页

### 2.2 文件搜索实现

**位置**：`SearchFilePaths()` 函数（第12416行）

**实现方式**：
```autohotkey
; 1. 直接调用 GetEverythingResults() 获取文件搜索结果
EverythingResults := GetEverythingResults(Keyword, everythingLimit + Offset + 1, true)

; 2. 返回完整的文件信息（Map格式）
result := Map()
result["Path"] := fullPath
result["IsDirectory"] := isDirectory
result["Type"] := isDirectory ? "folder" : "file"
result["Size"] := fileSize
result["DateModified"] := fileTime
result["Attributes"] := attributes

; 3. 转换为统一的结果格式，显示在SearchCenter的ListView中
```

**特点**：
- ✅ **直接搜索文件系统**：使用 Everything 搜索整个文件系统
- ✅ **返回完整信息**：路径、类型、大小、修改日期、属性
- ✅ **支持文件夹搜索**：`includeFolders := true`
- ✅ **独立结果展示**：文件结果作为独立的数据类型显示
- ✅ **支持分页**：通过 Offset 参数实现分页加载
- ✅ **结果丰富**：显示文件大小、修改日期等详细信息

## 三、ClipboardManagement 搜索能力

### 3.1 搜索架构

**单一数据源搜索**：
- 仅搜索剪贴板数据库（ClipMain表）
- 通过 Everything 搜索**过滤**数据库中的文件路径记录

### 3.2 文件搜索实现

**位置**：`RefreshClipboardListView()` 函数（第17816-17883行）

**实现方式**：
```autohotkey
; 1. 使用 Everything 搜索文件路径
DllCall(evDll "\Everything_SetSearchW", "WStr", SearchKeyword)
DllCall(evDll "\Everything_SetMax", "UInt", EverythingLimit)
DllCall(evDll "\Everything_QueryW", "Int", 1)

; 2. 获取文件路径列表
FilePaths := []
Loop Min(ResultCount, EverythingLimit) {
    ; 获取文件路径
    FilePathStr := StrGet(FilePathBuffer)
    FilePaths.Push("'" . StrReplace(FilePathStr, "'", "''") . "'")
}

; 3. 使用 SQL IN 子句过滤数据库记录
WhereClause .= " AND SourcePath IN (" . InClause . ")"
```

**特点**：
- ⚠️ **间接搜索**：只搜索数据库中已存在的文件路径记录
- ⚠️ **依赖SourcePath字段**：只能搜索剪贴板历史中记录的文件路径
- ⚠️ **功能受限**：无法搜索文件系统中未复制过的文件
- ✅ **回退机制**：Everything失败时使用SQL LIKE搜索
- ✅ **分类过滤**：支持按数据类型（Text/Code/Link/Image）过滤

## 四、能力对比分析

### 4.1 搜索范围对比

| 特性 | SearchCenter | ClipboardManagement |
|------|-------------|-------------------|
| **搜索范围** | 整个文件系统 | 仅数据库中的文件路径记录 |
| **数据来源** | Everything直接搜索 | Everything + SQL过滤 |
| **新文件发现** | ✅ 可以 | ❌ 不可以（必须先在剪贴板中） |
| **文件夹搜索** | ✅ 支持 | ❌ 不支持 |
| **文件信息** | ✅ 完整（大小、日期等） | ⚠️ 仅路径 |

### 4.2 搜索流程对比

**SearchCenter流程**：
```
用户输入关键词
    ↓
GetEverythingResults() 直接搜索文件系统
    ↓
返回文件列表（包含完整信息）
    ↓
转换为统一格式
    ↓
显示在ListView中
```

**ClipboardManagement流程**：
```
用户输入关键词
    ↓
Everything搜索文件路径
    ↓
获取路径列表
    ↓
SQL查询：WHERE SourcePath IN (路径列表)
    ↓
从数据库获取匹配的记录
    ↓
显示在ListView中
```

### 4.3 功能差异总结

#### SearchCenter的优势：
1. **直接文件搜索**：可以搜索整个文件系统，不依赖数据库
2. **完整文件信息**：显示文件大小、修改日期、类型等
3. **文件夹支持**：可以搜索和显示文件夹
4. **独立展示**：文件结果作为独立数据类型，与剪贴板数据分开显示
5. **分页支持**：支持Offset分页加载

#### ClipboardManagement的限制：
1. **搜索范围受限**：只能搜索数据库中已存在的文件路径
2. **依赖历史记录**：无法发现新的、未复制过的文件
3. **信息不完整**：只显示路径，没有文件大小、日期等信息
4. **无文件夹支持**：不搜索文件夹
5. **混合显示**：文件结果与剪贴板数据混在一起

## 五、改进建议

### 5.1 增强 ClipboardManagement 的搜索能力

#### 方案1：参考 SearchCenter 实现直接文件搜索

**改进点**：
- 添加独立的"文件搜索"标签页或分类
- 使用 `GetEverythingResults()` 直接搜索文件系统
- 显示完整的文件信息（大小、日期等）
- 支持文件夹搜索

**实现示例**：
```autohotkey
; 在RefreshClipboardListView中添加文件搜索分支
if (SearchKeyword != "" && CurrentCategory == "File") {
    ; 直接使用Everything搜索文件系统
    EverythingResults := GetEverythingResults(SearchKeyword, EverythingLimit, true)
    
    ; 转换为剪贴板管理格式
    for index, result in EverythingResults {
        if (Type(result) = "Map") {
            path := result["Path"]
            isDirectory := result["IsDirectory"]
            fileSize := result["Size"]
            dateModified := result["DateModified"]
            
            ; 创建文件结果项
            fileItem := Map()
            fileItem["Content"] := path
            fileItem["SourcePath"] := path
            fileItem["DataType"] := isDirectory ? "Folder" : "File"
            fileItem["Size"] := fileSize
            fileItem["DateModified"] := dateModified
            
            ClipboardItems.Push(fileItem)
        }
    }
}
```

#### 方案2：混合搜索模式

**改进点**：
- 保留现有的数据库过滤功能（搜索已复制的文件）
- 添加"文件系统搜索"选项，直接搜索整个文件系统
- 用户可以选择搜索模式：
  - "已复制文件"：仅搜索数据库中的文件路径
  - "全部文件"：搜索整个文件系统

**实现示例**：
```autohotkey
; 添加搜索模式选择
global ClipboardSearchMode := "Database"  ; "Database" 或 "FileSystem"

; 在搜索时根据模式选择
if (ClipboardSearchMode == "FileSystem") {
    ; 直接搜索文件系统
    EverythingResults := GetEverythingResults(SearchKeyword, EverythingLimit, true)
    ; ... 处理结果
} else {
    ; 使用现有的数据库过滤方式
    ; ... 现有代码
}
```

#### 方案3：增强现有实现

**改进点**：
- 在Everything搜索时获取更多文件信息
- 在SQL查询后补充文件信息（大小、日期等）
- 支持文件夹搜索

**实现示例**：
```autohotkey
; 在获取Everything结果时，同时获取文件信息
Loop Min(ResultCount, EverythingLimit) {
    ; 获取文件路径
    FilePathStr := StrGet(FilePathBuffer)
    
    ; 获取文件属性
    attributes := DllCall(evDll "\Everything_GetResultAttributes", "UInt", A_Index-1, "UInt")
    isDirectory := (attributes & 0x10) != 0
    fileSize := DllCall(evDll "\Everything_GetResultSize", "UInt", A_Index-1, "Int64")
    
    ; 存储完整信息
    FileInfo := Map()
    FileInfo["Path"] := FilePathStr
    FileInfo["IsDirectory"] := isDirectory
    FileInfo["Size"] := fileSize
    FilePaths.Push(FileInfo)
}

; 在SQL查询后，补充文件信息到结果中
for Index, Row in ResultTable.Rows {
    SourcePath := Row[4]  ; SourcePath字段
    
    ; 查找对应的文件信息
    for FileInfo in FilePaths {
        if (FileInfo["Path"] == SourcePath) {
            Row["FileSize"] := FileInfo["Size"]
            Row["IsDirectory"] := FileInfo["IsDirectory"]
            break
        }
    }
}
```

### 5.2 优先级建议

#### 🔴 高优先级（必须改进）

1. **支持文件夹搜索**
   - 当前只搜索文件，无法搜索文件夹
   - 参考SearchCenter的实现，添加文件夹支持

2. **获取文件详细信息**
   - 当前只显示路径，缺少大小、日期等信息
   - 使用Everything API获取完整文件信息

#### ⚠️ 中优先级（建议改进）

3. **直接文件系统搜索**
   - 添加独立选项，允许搜索整个文件系统
   - 不限于数据库中已存在的文件路径

4. **搜索模式选择**
   - 提供"数据库搜索"和"文件系统搜索"两种模式
   - 用户可以根据需求选择

#### 📝 低优先级（可选优化）

5. **结果排序优化**
   - 文件结果可以按大小、日期排序
   - 文件夹和文件分开显示

6. **搜索语法支持**
   - 支持Everything的高级搜索语法
   - 如：`ext:txt`, `size:>100MB` 等

## 七、总结

### 当前状态

**SearchCenter**：
- ✅ 功能完整，支持直接文件系统搜索
- ✅ 显示完整文件信息
